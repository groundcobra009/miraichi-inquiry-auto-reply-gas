# Google Workspace Studio プロンプト

## Gemini に相談

以下を Gemini ステップのプロンプトに貼り付ける。

```text
Googleフォームの最新回答を分析し、返信用シートに追加するためのJSONを作成してください。

【入力】
フォーム回答:
{{フォームの回答}}

【目的】
このJSONは次の「抽出」ステップで値を取り出し、Googleスプレッドシートの「返信用」シートに1行追加するために使います。

返信用シートの列は以下です。
A列: お名前
B列: メールアドレス
C列: 会社名
D列: 問い合わせ分類
E列: 問い合わせ内容
F列: 優先度
G列: 返信要否
H列: 返信件名
I列: 返信案
J列: チェック
K列: 返信
L列: 最終更新日時

【重要な出力ルール】
- 必ず有効なJSONだけを返してください。
- Markdown、説明文、コードブロック、前置き、後書きは一切不要です。
- JSONのキー名は必ず以下の12個だけにしてください。
- 値が不要な場合は null ではなく空文字 "" を入れてください。
- 返信案に改行を含める場合も、有効なJSON文字列として返してください。
- J列「チェック」は人間が確認してTRUEにする列なので、Geminiは必ず false を入れてください。
- K列「返信」はGASが送信後にTRUEにする列なので、Geminiは必ず false を入れてください。

【出力JSON形式】
{
  "name": "お名前",
  "email": "メールアドレス",
  "company": "会社名",
  "category": "問い合わせ分類",
  "inquiry_body": "問い合わせ内容",
  "priority": "高|中|低",
  "reply_required": "要返信|返信不要|要確認",
  "reply_subject": "返信件名。不要な場合は空文字",
  "reply_draft": "返信案。不要な場合は空文字",
  "approval_checked": false,
  "sent": false,
  "updated_at": "現在日時または空文字"
}

【元データの転記ルール】
- name はフォーム回答の「お名前」をそのまま入れてください。
- email はフォーム回答の「メールアドレス」をそのまま入れてください。
- company はフォーム回答の「会社名」をそのまま入れてください。
- category はフォーム回答の「問い合わせ分類」をそのまま入れてください。
- inquiry_body はフォーム回答の「問い合わせ内容」をそのまま入れてください。
- 元データに該当項目がない場合は空文字 "" にしてください。

【判定ルール】

1. priority
- 高:
  - 講演依頼
  - セミナー依頼
  - 研修依頼
  - AI相談
  - GAS相談
  - Google Workspace相談
  - 業務自動化相談
  - システム開発相談
  - 取材依頼
  - 登壇依頼
  - 具体的な仕事の依頼
  - こちらの支援につながる明確な相談
  - すぐ商談や打ち合わせにつながりそうな内容

- 中:
  - 詳細は不足しているが相談意図がある
  - 案件化の可能性がある
  - 協業や相談の可能性がある
  - 追加確認すれば判断できそうな内容
  - 返信不要とは言い切れない内容

- 低:
  - 営業メール
  - 売り込み
  - 営業代行
  - SEO営業
  - 広告営業
  - 一斉送信と思われる営業
  - 自社サービスを紹介したいだけの内容
  - こちらへの依頼ではなく、相手が営業したいだけの内容

2. reply_required
- 要返信:
  - 仕事の依頼
  - 講演、セミナー、研修、取材、登壇の依頼
  - AI、GAS、Google Workspace、業務自動化などの具体的な相談
  - こちらから前向きにプッシュして商談化したい内容

- 返信不要:
  - 一方的な営業
  - 売り込み
  - SEO営業、広告営業、営業代行
  - 一斉送信らしい内容
  - こちらに依頼したいのではなく、相手が商品やサービスを売りたいだけの内容

- 要確認:
  - 要返信か返信不要か判断が難しい
  - 内容が短すぎる
  - 相談意図はありそうだが、何を求めているか不明

3. reply_subject
- reply_required が「要返信」の場合は自然な日本語のメール件名を作成してください。
- reply_required が「返信不要」の場合は空文字 "" にしてください。
- reply_required が「要確認」の場合は「お問い合わせ内容の確認」にしてください。

4. reply_draft
- reply_required が「要返信」の場合だけ、日本語で丁寧かつ前向きな返信案を作成してください。
- 構成は「挨拶」「内容確認」「不足情報の確認」「結び」にしてください。
- 名前が分かる場合は「〇〇 様」から始めてください。
- 不足情報の確認は、実施希望時期、目的や背景、対象者や参加人数、相談したい具体的な内容、予算感、打ち合わせ可能な日程、オンライン/対面の希望から2〜4個だけ聞いてください。
- reply_required が「返信不要」の場合、reply_draft は空文字 "" にしてください。断り文は作成しないでください。
- reply_required が「要確認」の場合、短い確認用の返信案を作成してください。

5. approval_checked
- 必ず false にしてください。
- 人間がJ列をTRUEにした場合だけGASが送信対象にします。

6. sent
- 必ず false にしてください。
- GASがメール送信に成功した後、K列をTRUEにします。

7. updated_at
- 現在日時を取得できる場合は "yyyy/MM/dd HH:mm:ss" 形式で入れてください。
- 現在日時を取得できない場合は空文字 "" にしてください。
```

## 抽出ステップ

Gemini の出力JSONから以下のキーを抽出する。

| JSONキー | 返信用シート列 |
| --- | --- |
| `name` | A列 お名前 |
| `email` | B列 メールアドレス |
| `company` | C列 会社名 |
| `category` | D列 問い合わせ分類 |
| `inquiry_body` | E列 問い合わせ内容 |
| `priority` | F列 優先度 |
| `reply_required` | G列 返信要否 |
| `reply_subject` | H列 返信件名 |
| `reply_draft` | I列 返信案 |
| `approval_checked` | J列 チェック |
| `sent` | K列 返信 |
| `updated_at` | L列 最終更新日時 |

## 行を追加

「返信用」シートに新しい行を追加する。

列の対応:

- A列: `name`
- B列: `email`
- C列: `company`
- D列: `category`
- E列: `inquiry_body`
- F列: `priority`
- G列: `reply_required`
- H列: `reply_subject`
- I列: `reply_draft`
- J列: `approval_checked`
- K列: `sent`
- L列: `updated_at`

人間が返信内容を確認して問題なければ、J列「チェック」をTRUEにする。GASはJ列がTRUE、K列がFALSEまたは空欄、G列が「要返信」の行だけ送信する。
